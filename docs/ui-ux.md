# Jenkins-钉钉加密转发器 — UI/UX 设计文档

> 版本：1.0 | 更新日期：2026-06-18 | 设计师：颜好看

---

## 1. 概述

本系统是部署在 DMZ 区的 Jenkins-钉钉加密转发器，核心为后端转发服务，附带一个 Web 管理面板用于运维监控与配置管理，同时需要设计钉钉 OA 审批表单。

设计目标：**简洁、功能导向、信息密集**。这是一个技术工具，面向运维与开发人员，不做过度装饰，以信息可读性和操作效率为第一优先级。

## 2. 竞品 UI 分析

## 3. 设计方向与对标品牌

### 3.1 对标品牌

综合调研，选择以下两个品牌作为设计对标：

| 对标品牌 | 借鉴要点 |
|---------|---------|
| **Vercel Dashboard** | 暗色模式优先、极简开发者美学、状态无处不在、功能性空状态、乐观 UI、Geist 字体 |
| **Linear** | 深色工程美学、信息密度与留白平衡、Inter 字体、Indigo 品牌色、秩序感与工作流组织 |

### 3.2 设计原则

基于 Vercel 和 Linear 的设计哲学，结合本系统运维工具属性，确立以下设计原则：

1. **暗色优先** — 运维人员长时间在终端/IDE 的暗色环境中工作，暗色界面减少视觉疲劳和上下文切换。提供亮色模式作为可选项。
2. **状态无处不在** — 构建状态、审批状态、连接状态应在不切换页面的情况下即可感知（标签页图标、状态徽章、时间线指示器）。
3. **信息密度优先** — 运维面板需要在一屏内展示尽可能多的有效信息，避免大面积留白和装饰性元素。
4. **功能性空状态** — 空状态不是插图，而是操作指引。展示下一步该做什么（如"尚未配置 Jenkins 连接，前往配置"）。
5. **即时反馈** — 采用乐观 UI 和骨架屏，避免加载转圈。操作响应应在 100ms 内体现。
6. **等宽字体用于数据** — 日志、构建号、时间戳、ISSUE 号等技术数据使用等宽字体，便于扫读和对齐。
7. **不过度设计** — 这是工具不是产品展示页。每个元素都应服务于功能，去除一切装饰性动效和插图。

## 5. 字体方案

### 5.1 字体族

| Token | 字体栈 | 用途 |
|-------|-------|------|
| `--font-sans` | `'Inter', -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei', sans-serif` | UI 界面、正文、标题 |
| `--font-mono` | `'JetBrains Mono', 'Geist Mono', 'SF Mono', 'Consolas', monospace` | 代码、日志、技术数据 |
| `--font-sans-cn` | `'PingFang SC', 'Microsoft YaHei', 'Inter', sans-serif` | 中文优先场景 |

**说明**：
- 英文/数字使用 **Inter**（与 Linear 一致），中文回退到 PingFang SC（macOS）/ Microsoft YaHei（Windows）
- 等宽字体使用 **JetBrains Mono**（开发者熟悉，字符辨识度高），用于日志流、构建号、ISSUE 号、命令等
- 不引入 Web Font 文件，使用系统已安装字体，保证加载速度

| Token | 值 | 用途 |

暗色模式以边框代替阴影（边框即层级），辅以微弱阴影：

| Token | 值 | 用途 |
|-------|----|------|
| `--shadow-0` | none | 扁平，无阴影 |
| `--shadow-1` | `0 0 0 1px var(--border-default)` | 边框环，卡片默认 |
| `--shadow-2` | `0 0 0 1px var(--border-default), 0 1px 2px rgba(0,0,0,0.4)` | 悬停卡片 |

### 6.4 动画/过渡

| Token | 值 | 用途 |
|-------|----|------|
| `--transition-fast` | `100ms ease` | 按钮态切换、颜色变化 |

| `--transition-slow` | `250ms ease` | 页面过渡 |

**原则**：过渡时间不超过 250ms，运维工具追求即时反馈，避免冗长动画。日志流不使用动画，直接追加。

## 8. 钉钉审批表单设计

### 8.1 钉钉表单技术约束

根据钉钉 OA 审批平台的技术特性：

- 表单在钉钉管理后台设计，通过 API 创建/更新模板
- 支持的控件类型：单行文本、多行文本、下拉选择、日期、金额、联系人等
- 表单字段以**中文标题**为唯一标识，不可重复
- 单行文本传入 URL 会自动转为可点击链接
- 钉钉客户端列表仅展示表单前三个字段
- 审批人通过联系人组件或流程节点配置指定

### 8.2 表单字段设计

本系统钉钉审批表单包含四个核心业务字段，加上辅助字段：

#### 表单模板信息

| 属性 | 值 |
|------|-----|
| 模板名称 | Jenkins 构建审批 |
| 模板描述 | 提交 Jenkins 构建请求，经审核员审批后触发构建 |
| 分组 | 研发运维 |

#### 字段清单

| 序号 | 字段标题 | 控件类型 | 必填 | 说明 |
|------|---------|---------|------|------|
| 1 | ISSUE 工单号 | 单行文本 | 是 | 工单编号，便于追踪关联 |
| 2 | Project 项目 | 单行文本 | 是 | Jenkins 项目名称 |
| 3 | Branch 分支 | 单行文本 | 是 | 代码分支名 |
| 4 | Build 构建命令 | 多行文本 | 是 | 构建脚本/命令 |
| 5 | 构建说明 | 多行文本 | 否 | 补充说明（可选） |
| 6 | 转发器记录号 | 单行文本 | 否 | 系统自动填充（隐藏字段） |

#### 字段详细设计

**字段 1：ISSUE 工单号**

| 属性 | 值 |
|------|-----|
| 控件类型 | 单行文本（TextField） |
| 标题 | ISSUE 工单号 |
| 占位提示 | 请输入工单号，如 ISSUE-245 |
| 必填 | 是 |
| 输入校验 | 正则 `^[A-Z]+-\d+$`，大写字母+横线+数字 |
| 字符限制 | 最大 50 字符 |
| 说明文字 | 填写 Jira/工单系统中的 ISSUE 编号 |

**字段 2：Project 项目**

| 属性 | 值 |
|------|-----|
| 控件类型 | 单行文本（TextField） |
| 标题 | Project 项目 |
| 占位提示 | 请输入 Jenkins 项目名，如 web-frontend |
| 必填 | 是 |
| 字符限制 | 最大 100 字符 |
| 说明文字 | Jenkins 中对应的 Job/Project 名称 |

> **设计考量**：不使用下拉选择，因为项目列表动态变化且可能较多。如需下拉，可通过业务控件或自定义 Setter 从 Jenkins API 动态拉取项目列表。MVP 阶段使用单行文本，后期可升级为动态下拉。

**字段 3：Branch 分支**

| 属性 | 值 |
|------|-----|
| 控件类型 | 单行文本（TextField） |
| 标题 | Branch 分支 |
| 占位提示 | 请输入分支名，如 feature/login |
| 必填 | 是 |
| 字符限制 | 最大 200 字符 |
| 输入校验 | 禁止空格和中文字符（分支命名规范） |
| 说明文字 | 代码仓库分支名称，如 main / develop / feature-xxx |

**字段 4：Build 构建命令**

| 属性 | 值 |
|------|-----|
| 控件类型 | 多行文本（Textarea） |
| 标题 | Build 构建命令 |
| 占位提示 | 请输入构建命令，如 npm run build |
| 必填 | 是 |
| 字符限制 | 最大 2000 字符 |
| 行数 | 默认展示 4 行 |
| 说明文字 | Jenkins 构建执行的命令或脚本，支持多行 |

### 8.3 审批流程设计

#### 流程节点

```
发起人提交
    │
    ▼
┌─────────────┐     拒绝      ┌──────────┐
│ 审核员 A     │ ──────────────▶│ 流程结束  │
│ (第一位审批) │               │ (构建不执行)│
└──────┬──────┘               └──────────┘
       │ 通过
       ▼
┌─────────────┐     拒绝      ┌──────────┐
│ 审核员 B     │ ──────────────▶│ 流程结束  │
│ (第二位审批) │               │ (构建不执行)│
└──────┬──────┘               └──────────┘
       │ 通过
       ▼
┌─────────────┐     拒绝      ┌──────────┐
│ 审核员 C     │ ──────────────▶│ 流程结束  │
│ (第三位审批) │               │ (构建不执行)│
└──────┬──────┘               └──────────┘
       │ 通过
       ▼
┌──────────────────────────────┐
│ 转发器自动触发 Jenkins 构建    │
│ (系统节点，无需人工)           │
└──────────────────────────────┘
```

#### 审批人配置

- 三位审核员依次审批（串行会签）
- 任一审核员拒绝，流程终止，不触发构建
- 三位全部通过后，转发器收到回调，触发 Jenkins 构建
- 审核员通过钉钉联系人组件指定，或在流程设计中固定为指定人员

#### 二次审核流程

当 Jenkins 构建过程中检测到敏感文件修改（以 `special.md` 创建为 demo）：

```
Jenkins 构建中
    │
    ▼ 检测到 special.md 创建
┌──────────────────────────────┐
│ 转发器发送二次审核请求到钉钉    │
│ (复用审批表单，标记为二次审核)  │
└──────────────────────────────┘
    │
    ▼
┌─────────────┐
│ 审核员审批    │
│ (确认是否继续)│
└──────┬──────┘
       │ 通过
       ▼
┌──────────────────────────────┐
│ 转发器继续/重新触发 Jenkins 构建│
└──────────────────────────────┘
       │ 拒绝
       ▼
┌──────────────────────────────┐
│ 终止构建，通知发起人           │
└──────────────────────────────┘
```

### 8.4 钉钉表单呈现建议

钉钉审批表单在钉钉客户端（移动端/PC 端）中呈现，需注意：

1. **字段顺序**：ISSUE 工单号、Project 项目、Branch 分支、Build 构建命令 — 钉钉列表仅展示前三个字段，恰好显示 ISSUE/Project/Branch，便于列表中快速识别
2. **只读展示**：审批详情页中四个字段全部只读展示，审批员不可修改发起人填写的内容
3. **构建说明**：可选字段，用于发起人补充上下文，帮助审批员决策
4. **转发器记录号**：隐藏字段，系统自动填充转发器内部的记录 ID，用于数据关联，不展示给用户
5. **审批意见**：每位审核员审批时填写审批意见（钉钉原生功能，无需额外字段）

### 8.5 构建结果回传

构建完成后，转发器将结果回传钉钉：

- **构建成功**：在审批单中添加评论（"构建 #128 成功，耗时 4m32s"），审批流程自动关闭
- **构建失败**：在审批单中添加评论（"构建 #128 失败：编译错误，详见日志"），通知发起人
- **二次审核**：若触发二次审核，在审批单中添加评论说明原因（"构建中检测到敏感文件 special.md，需二次审核"）

## 9. 交互设计原则

### 9.1 实时性

- 构建状态、审批状态变更通过轮询（5 秒间隔）或 WebSocket 实时更新
- 请求日志页面使用 WebSocket/SSE 流式推送，无轮询
- 进行中的操作使用乐观 UI，先更新界面再等待服务端确认

### 9.2 状态感知

- 浏览器标签页图标（favicon）反映全局状态：有进行中的构建/审批时显示动态图标
- 页面标题前缀显示进行中数量：`(2) JD-Relay 仪表盘`
- 所有状态变化使用状态徽章，颜色严格遵循配色方案中的语义色

### 9.3 错误处理

- 错误提示使用 Toast 通知，红色背景，自动消失（5 秒）
- 网络断开时顶栏显示离线提示条
- 构建失败在列表中红色高亮，点击可查看错误日志
- 配置错误（如连接失败）在对应配置项下方红色文字提示

### 9.4 加载状态

- 列表加载使用骨架屏（Skeleton），不使用转圈
- 数据更新使用 SWR 策略，后台静默刷新
- 按钮提交使用按钮内 loading 态，禁用防重复提交

### 9.5 空状态

空状态为功能性引导，不使用装饰插图：

| 场景 | 空状态文案 |
|------|-----------|
| 无审批记录 | "暂无审批记录。在钉钉中发起 Jenkins 构建审批后将显示在这里。" |
| 无构建记录 | "暂无构建记录。审批通过后将自动触发构建。" |
| 无日志 | "暂无日志。系统运行后将在此实时显示请求日志。" |
| 未配置 Jenkins | "Jenkins 尚未配置。前往 [配置管理] 设置 Jenkins 连接。" |
| 未配置钉钉 | "钉钉尚未配置。前往 [配置管理] 设置钉钉连接。" |

### 9.6 响应式

- 最小支持宽度 1024px（运维面板主要在桌面端使用）
- 1024px-1280px：侧边栏可折叠，统计卡片 2 列
- 1280px 以上：侧边栏展开，统计卡片 4 列
- 不专门适配移动端（运维操作不在移动端进行；钉钉端审批由钉钉 App 原生呈现）
